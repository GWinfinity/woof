## P0 中缺失的核心规则 (15条)

### Tier 1: 基础错误（5条缺失）

#### 1. SA5009: Printf 格式与参数不匹配 ⚠️ CRITICAL
```go
// 错误示例
fmt.Printf("%d", "string")  // %d 期望整数，得到字符串
fmt.Printf("%s")           // 缺少参数
fmt.Printf("%s %s", "arg") // 参数不足
```

**为什么属于P0：**
- **严重性**: 运行时 panic（高）
- **可实现性**: AST 可检查（分析格式字符串和参数数量）
- **普适性**: 所有使用 fmt 包的项目（99%）
- **误报率**: < 1%（明确的 API 契约）
- **常见度**: 极高，是初学者最常犯的错误之一

**实现复杂度**: 中等（需要解析 Printf 格式动词）

### Tier 2: 并发安全（4条缺失）

#### 6. F831: 循环变量在 Goroutine 中捕获 ⚠️⚠️ CRITICAL
```go
// 错误示例 - 这是 Go 最臭名昭著的 Bug 之一！
for _, val := range values {
    go func() {
        fmt.Println(val)  // 所有 goroutine 都打印最后一个值！
    }()
}

// 正确做法
for _, val := range values {
    go func(v string) {
        fmt.Println(v)
    }(val)
}
```

**为什么属于P0：**
- **严重性**: 数据竞争，逻辑错误（极高）
- **可实现性**: 可检查（检测循环内的 go func() 使用循环变量）
- **普适性**: 所有使用 goroutine 的项目（70%）
- **误报率**: 低
- **常见度**: **极高** - 这是 Go 最著名的坑之一！
- **独特性**: Go 1.22 之前非常容易出错，即使 Go 1.22 修复了，也需要兼容旧代码

**实现复杂度**: 中等（需要检测变量捕获）

**为什么这个特别重要：**
这是 Go 社区公认的**最常见并发 Bug**，甚至在 Go 官方文档中都特别警告。在 Go 1.22 之前，这是几乎每个 Go 开发者都会踩的坑。

#### 7. SA2000: WaitGroup.Add 在 Goroutine 中调用 ⚠️ CRITICAL
```go
// 错误示例
var wg sync.WaitGroup
for i := 0; i < 10; i++ {
    go func() {
        wg.Add(1)        // 错误：应该在 go 语句之前调用
        defer wg.Done()
        doWork()
    }()
}
wg.Wait()  // 可能Wait时还有goroutine没Add
```

**为什么属于P0：**
- **严重性**: 数据竞争，Wait 可能提前返回（高）
- **可实现性**: 可检查（检测 go 语句内的 wg.Add）
- **普适性**: 使用 WaitGroup 的项目（50%）
- **误报率**: 极低
- **常见度**: 中等

**实现复杂度**: 简单（已在 P0 中实现，确认一下）

**状态**: ✅ 已实现

#### 9. B003: Goroutine 泄露 ⚠️ HIGH
```go
// 错误示例 - Fire and Forget 可能导致泄露
func process(ctx context.Context) {
    go func() {
        // 长时间运行的 goroutine
        for {
            select {
            case <-ctx.Done():
                return  // 正确：响应取消
            }
        }
    }()
}

// 错误版本 - 不响应 ctx
go func() {
    for {
        doWork()  // 永远不会停止！
    }
}()
```

**为什么属于P0：**
- **严重性**: 资源泄露，内存/CPU 耗尽（高）
- **可实现性**: 可检查（检测没有退出条件的 goroutine）
- **普适性**: 使用 goroutine 的项目（70%）
- **误报率**: 中等（需要启发式）
- **常见度**: 中高

**实现复杂度**: 中等（需要控制流分析）
